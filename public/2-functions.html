Lesson 2: Functions

How To Save Actions

When it comes to actually doing something with programming, you're going to need a Function. A function is simply a reusable chunk of code that carries out some action. You can put any code you want into a function, and you write them like this: 

// First, we define it
const square = (number) => {
  return number * number
}

// then we call it!
square(4)
// This gives us 16

Or, you can use pre-defined functions provided by the language itself. Just like how JavaScript gives us keywords like let or const, it also gives us a lot of built in functions to use. Let's start by logging something to our console using the console object's log function (using the dot notation we just learned about)! 

// We can log raw values
console.log('Hello there!')
console.log(12)
console.log(true)

// we can also log variables
let city = 'New York City'
console.log(city)

// Print multiple values by using commas
console.log('I live in', city)

Calling Functions

Before learning how to define our own functions, let's go over how to use some of the built in JS functions. In order to call (or "run") a function, you add a pair of () onto the end of the function's name. If the function needs extra pieces of information, called Arguments, you put them in between the ().

console.log in depth

The console.log function takes any number of arguments and then logs each one to the screen. 

// This is the function name
console.log

// we call it by adding ()
console.log()

// finally, let's add an argument
console.log('Hello!')

// Comma separated Arguments get
// printed with spaces in between
console.log('Hello', "there", "friend")

The ability to log multiple things makes debugging, or fixing, your programs a lot easier. By using a string argument as a label, you can pass in a variable as a second argument to make sure it is what you think it's supposed to be. 

let city = undefined
console.log('City:', city)

// Thanks to our log, we can easily see
// that variable is wrong
// can you please fix it?

To run this codepen, make sure you're connected to the internet. Debugging Practice

The label is helpful because if a value is ever undefined, console.log may not print anything. Adding a label makes sure something is always printed. Plus, the labels keep things organized.

Defining Functions

Ok now that we know how to call existing functions, let's learn how to write our own. There are basically 4 parts to any function:
  • Name: How we call the function
  • Parameters: The Argument definitions
  • Body: The code that gets run with each call
  • Return Value: The data we get out of a function

Breaking it all down

As you can see, the name is a lot like our last lesson. That's on purpose. A variable name can point that line to anything. It could be a piece of data, but it could also be a function. If a variable points to a function, we just call it a function. You don't have to say "my function variable" or anything.

The New Hotness

There's an older way to define a function using a special function keyword, but it has some secret baggage we don't want to deal with right now. So, we won't use it yet. But, in case you see it in online tutorials, here's how to convert it into the modern Arrow Function syntax. 

function oldFunction (myParam) {
  return myParam
}

// Which is the same as this
const newFunction = (myParam) => {
  return myParam
}

Fun fact, it's called an arrow function because we use a little arrow (=>) to define them.

Let's take a closer look at each part of the function.

The Name

Notice that I'm using const instead of let. Since our variables might need to change, we just stick with let for simplicity. But a function assignment should never change. Think of it like easing into using let and const like the pros.

The Parameters

If we pass values into our called function, we refer to them as Arguments, but when we're dealing with those values in our function's definition, we call them Parameters. Think of Parameters like mini variables that we only use in our function. 

// Here our parameters are
// num1 and num2
const add = (num1, num2) => {
  return num1 + num2
}

// which match the arguments 2 and 4
add(2,4)

Parameters go inside the (). When there's more than one you separate them out with commas. If there are no parameters, you still need to use () when defining and calling the function. 

const greet = () => {
  console.log("Hey!")
}

greet()

Parameters can be any data type, but they only exist inside the function. 

const add = (num1, num2) => {
  return num1, num2
}

num1
NO! BAD! THAT WILL BREAK!

What if you want a parameter value outside of a function? You use that function's return value.

The Function Body

Anything within the {} is the body of the function. When we call a function, it's the code that runs. On the flip side, it won't do anything until you call it.

Spacing for functions

To tell what is or isn't inside a function, always indent one level (that means hit the tab key on your keyboard once). Most editors will automatically do this for you when you press enter.

The Return Value

Sometimes we want to use a function to create a value and, instead of logging it, save it to a variable. In order to get values out of our functions, we need to use the return keyword. Look how we use add() to set the value of the oneHundred variable. 

const add = (num1, num2) => {
  return num1 + num2
}

console.log(add(4,5))

let oneHundred = add(50,50)
console.log(oneHundred)

The value is now saved to a variable, so we will have access to it without having to re-call the function. If we just did console.log(add(50,50)), we'd have to keep calling add() every time we wanted 100. By returning a value, we can call add() once, and reference the stored variable instead.

Once you use return that's it, that's the end of the function. Any code written after return simply wont run. 

const testReturn = () => {
  console.log("I run!")
  return true
  console.log("I do not!")
}

testReturn()

Implicit vs Explicit Returns

When you do something "explicitly" it means "to do it obviously" The opposite is to do something "implicitly", which means it's "not directly shown."

Arrow functions can explicitly return a value using the return keyword. However, if your function is only one line long, you can implicitly return a value by removing the {}. 

// explicit return
const add = (num1, num2) => {
  return num1 + num2
}

// implicit return
const add = (num1, num2) => num1 + num2

I'm only bringing it up because it's a popular trick, and I don't want you to get confused if you see it in some other tutorials. Don't panic, they didn't forget the {}, the body is just one line long.

Common Errors

One last thing before we move on, I just want to mention some common errors that you should watch out for.

Call the function

Don't forget the () to call a function! 

const sayHi = () => {
  return "Hello there!"
}

let greeting = sayHi // WRONG

That code will just set the value of greeting as the function definition. If you log it, it looks like "[Function: sayHi] native code". Always call your function with ()!

Missing arguments

You would think that if you forget an argument when calling a function JavaScript would automatically break. It doesn't. Instead, it treats any missing arguments as if they had a value of undefined. 

const add = (num1, num2) => num1 + num2

let total = add(1)
// this logs NaN

Having something accidentally be undefined does weird things. For example, a number mixed with undefined will become NaN, which means "Not a Number". Always make sure that you pass in the right arguments!

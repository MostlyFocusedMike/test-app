Interview Questions

Do your best!

Here we are, the final questions! This might seem like a lot, but just remember, everything you need to know was in the lessons you just took. And before you get started, we just want you to know we're proud of you for making it this far. You have officially begun your journey to becoming an engineer.

Remember, we are not looking for perfection; we are looking for your best effort and problem solving skills. We do not expect you to get every question right, just attempt as many as you can.

How to submit

Below are 5 challenges, and each challenge describes one function. Your task is to add working versions of these functions into a single Replit project (like you learned about in Lesson 0). Each function's name should match the prompt so we know which is which. Please do not submit several Replits, all functions should be in one Replit. And be sure all code is inside a function, don't just submit random lines of code.

Once you're done, copy the Replit link into The Application Submission Form.

Please see the FAQ section at the very bottom of this page for more information. Good luck everyone!


Question 1: greet()

Write a function greet that takes a String argument, name. The function should RETURN the string "Hello, [DYNAMIC-NAME], how are you?"

Argument Info
Name Type Description
name String The person's name

Example Code

When you enter this code:

console.log(greet('Tom'))
console.log(greet('Jerry'))

You should see this in the console:

Hello Tom, how are you?
Hello Jerry, how are you?

Question 2: getRating()

Write a function getRating that takes 2 arguments: a String argument, movieTitle, and a String argument, review. The function should LOG two lines. On the first line should be either asterisks or "???", depending on the review passed in. The second line should be a string "Movie: [DYNAMIC-MOVIE-TITLE]".

The function has no return value.

Here's how to decide what stars to show:
  • 'bad' or 'awful': *
  • 'good': **
  • 'great': ***
  • Literally anything other string: '???'
Argument Info
Name Type Description
movieTitle String The title of the movie
review String A single word opinion on the film

Example Code

Below are some functions and their output

getRating('Cats', 'bad')
*
Movie: Cats
getRating('Cars', 'good')
**
Movie: Cars
getRating('Gravity', 'odd')
???
Movie: Gravity

Question 3: hideAndSeekCounter()

Write a function hideAndSeekCounter that takes a Number argument, endNum. The function should LOG several strings. First, it should count up to the given number starting from 1, with "..." after each number. But on the final number it should add "!" instead. Finally, one last line of "Ready or not, here I come!"

See the code examples below.

Argument Info
Name Type Description
endNum Number A number greater than 0

Example Code

hideAndSeekCounter(3)
1...
2...
3!
Ready or not, here I come!
hideAndSeekCounter(6)
1...
2...
3...
4...
5...
6!
Ready or not, here I come!
hideAndSeekCounter(1)
1!
Ready or not, here I come!

Question 4: cheer()

Write a function cheer that takes a String argument, word.

The function should LOG several strings. For every letter of word make a line print: "Give me a [UPPERCASE-LETTER]!", where the letter has been set to uppercase. Once the entire word has been shouted, print the line "What's it spell?" and then one last line, where you repeat the word three times, each with an exclamation point and all uppercase.

See the code example below.

Argument Info
Name Type Description
word String A lowercase word with no spaces or punctuation

Example Code

cheer('win')
Give me a W!
Give me a I!
Give me a N!
What's it spell?
WIN! WIN! WIN!
cheer('defense')
Give me a D!
Give me a E!
Give me a F!
Give me a E!
Give me a N!
Give me a S!
Give me a E!
What's it spell?
DEFENSE! DEFENSE! DEFENSE!

Question 5: betterCheer()

FAQs

Tap or click open any question that you'd like to learn more about

What would an example answer look like?

We basically just want the function definition. So if we asked you to define a function called add, in the Replit project it would look like this: 

const add = (num1, num2) => {
  return num1 + num2
}

You don't have to call the function or anything, though it's ok if you do for testing. This is also fine: 

const add = (num1, num2) => {
  return num1 + num2
}

console.log(add(4,5))

All that matters is that you have the functions defined so that we can call them when grading.

What if we don't complete every function?

That's ok! Just try to get through as many as you can. We would definitely like you to try to complete at least one though. The important part is that you are able to talk to your interviewer about your code.

You can also submit partially completed functions if you know how to do a certain part, but not the whole thing. We just want to see how you think and approach problems.

How will I be graded

There are some specific things we're looking for, like properly defined functions and return values, but that's not the main point. Our real interest is talking with you about how you learn and problem solve. Were there parts of the lessons you found interesting? Confusing? The whole interview process is designed to see how you think things through.

What are the next steps?

After you fill out the submission form, we'll schedule an interview with you. The interviews will have 2 sections: a general and technical. In the general section we'll talk about your coding journey so far, how you like to study, and other things about you as a person. In the technical section, a developer will talk with you about your code and go over your submission. This part is all about how well you communicate your technical ideas and how well you can collaborate.

No new coding challenges will be given to you. The most we might do is repair broken functionality in your submission or go over how we might refactor or add new features to an existing challenge. This is not a white boarding challenge, and at no point will we spring an algorithm question on you. We're going to be talking about coding in a low stress environment.
